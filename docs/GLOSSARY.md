# Glossary (SOC 2 Terms)

- **SOC 2**: Security audit framework evaluating controls for Trust Service Criteria.
- **Type I**: Point-in-time audit of control design.
- **Type II**: Period audit (e.g., 3–12 months) of control design + operating effectiveness.
- **Readiness**: Pre-audit assessment of preparedness, gaps, and expected costs/timeline.
- **Trust Service Criteria**: Security, Availability, Confidentiality, Processing Integrity, Privacy.
- **Readiness Score**: 0–100 normalized score representing typical audit readiness patterns.
- **Cost Range**: Estimated auditor + tooling + internal effort band produced by scoring engine.
- **Audit Timeline**: Months until planned audit; influences score and urgency.
- **Evidence**: Documentation and artifacts proving controls exist and operate.
- **Controls**: Policies/processes/technical measures required for SOC 2.
- **RLS (Row Level Security)**: Database mechanism controlling row access per role.
- **Service Role**: Supabase privileged key bypassing RLS; server-only.
- **Honeypot**: Hidden field (`website`) used to detect bots/spam.
